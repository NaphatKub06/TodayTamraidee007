1. จุดประสงค์ของโปรแกรม (Objectives)
  1.1.	เพื่อบันทึกและจัดเก็บข้อมูล: เพื่อให้ผู้ใช้งานสามารถจดบันทึกเหตุการณ์ประจำวัน (Daily Journal) ในรูปแบบดิจิทัล แทนการจดลงสมุด ซึ่งป้องกันการสูญหายและจัดเก็บได้อย่างเป็นระบบ
  1.2.	เพื่อความสะดวกในการเข้าถึงและใช้งาน: พัฒนาในรูปแบบ Web Application ทำให้ผู้ใช้งานสามารถล็อกอินเพื่อเขียนหรืออ่านบันทึกได้จากทุกที่ผ่านเบราว์เซอร์
  1.3.	เพื่อรักษาความเป็นส่วนตัว: มีระบบยืนยันตัวตน (Authentication) แยกบัญชีผู้ใช้งานแต่ละคนอย่างชัดเจน ข้อมูลของใครของมัน ไม่ปะปนกัน
  1.4.	เพื่อการบริหารจัดการข้อมูล: ผู้ใช้สามารถ สร้าง (Create), อ่าน (Read), แก้ไข (Update), และลบ (Delete) รายการบันทึกของตนเองได้อย่างอิสระ
2. การทำงานของโปรแกรม (System Workflow)
  2.1.	การสมัครสมาชิก (Registration):
    o	ผู้ใช้กรอกชื่อ (Username) และรหัสผ่าน
    o	ระบบตรวจสอบในฐานข้อมูลว่า Username นี้ซ้ำหรือไม่
    o	หากไม่ซ้ำ ระบบจะเข้ารหัสรหัสผ่าน (Password Hashing) และบันทึกลงฐานข้อมูล
  2.2.	การเข้าสู่ระบบ (Login):
    o	ผู้ใช้กรอก Username และ Password
    o	ระบบตรวจสอบความถูกต้อง หากถูกต้องจะสร้าง Session เพื่อยืนยันตัวตนและพาเข้าสู่หน้าหลัก
  2.3.	การจัดการบันทึก (Manage Entries):
    o	หน้าหลัก (Dashboard): ระบบดึงข้อมูลบันทึกทั้งหมดที่เป็นของ User นั้นๆ มาแสดงผลเรียงตามวันที่ล่าสุด
    o	การเพิ่มบันทึก: ผู้ใช้กรอกหัวข้อและรายการย่อย (List items) ระบบจะแปลงรายการย่อยเป็นรูปแบบ JSON และบันทึกลงฐานข้อมูล
    o	การแก้ไข/ลบ: ผู้ใช้เลือกบันทึกเก่าเพื่อแก้ไขข้อความ หรือเพิ่ม/ลดรายการย่อยได้ รวมถึงลบบันทึกที่ไม่ต้องการออกจากระบบ
  2.4.	การออกจากระบบ (Logout):
    o	ระบบจะทำลาย Session ของผู้ใช้งาน และพากลับไปยังหน้า Login
